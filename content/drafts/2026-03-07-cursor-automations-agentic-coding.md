## HEADLINE
A major AI coding platform launched event-driven automations that run continuously without human prompting — triggered by commits, Slack messages, or alerts — shifting engineers from task initiators to checkpoint reviewers.

## KEY FACTS
- Automations triggered by external events (code commits, PagerDuty alerts, timers, Slack messages) — no human prompt required to start a task
- MCP (Model Context Protocol) powers the incident response automation in production: alerts trigger an agent that queries live server logs in real time
- Platform is currently running hundreds of automations per hour; $2 billion ARR, doubled over the past three months
- Source credibility: Production deployment at scale; named engineering lead on record; $2B ARR confirmed by Bloomberg

## GREY AI ANGLE
The "humans as checkpoints, not initiators" model being built in software development is the organizational template for every knowledge-work function. MCP is the connective tissue making it possible — this is the most concrete live example of the protocol in production. For teams thinking about AI workflow design, Cursor Automations is the reference architecture: what it looks like when AI agents are woven into the operational fabric rather than layered on top of it.
