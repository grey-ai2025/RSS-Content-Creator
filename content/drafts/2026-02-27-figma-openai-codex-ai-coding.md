Figma just integrated two competing AI coding models in the same week. That is not a product decision — it is a strategic one.

First Claude Code. Then OpenAI's Codex. Both embedded directly inside the design workspace, both capable of generating front-end code from the design assets already sitting in the file.

**The handoff is the friction.** Design-to-development handoffs have historically been where intent gets lost — redlines misread, specs misinterpreted, rework cycles baked in before a single line of code ships. AI coding agents embedded directly in Figma don't just speed up that step. They eliminate the context switch that caused the errors in the first place.

**The model-agnostic play is the signal.** Figma is not betting on one AI coding model winning. It is betting on being the platform layer that survives regardless of which model wins — a position that accumulates value as the underlying AI tools keep improving. Think of it less like a software update and more like infrastructure: the platform where design lives becomes the platform where code gets born.

**The platform layer is where durable value accumulates.** Benchmark wars between models are real, but they are also temporary — the best model changes every few months. Platform integrations are stickier. For product teams evaluating AI coding tools, watching where integrations land is as important as watching which model scores highest.

The design-to-code pipeline is being restructured. Figma's move suggests the platform sitting in the middle of that pipeline intends to own it. 🎨
